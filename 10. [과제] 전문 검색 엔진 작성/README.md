#  10장. [과제] 전문 검색엔진 작성

## **강의 27: 과제 하테나 북마크 전문 검색 만들기**

### **목표**

- 하테나 북마크 **최근 1만 엔트리**를 대상으로 검색어를 포함하는 엔트리를 반환하는 **전문 검색엔진을 직접 만들어보는 과제**
- 역 인덱스를 활용하는 실전 연습

### **데이터 구성**

- 각 엔트리는 eid, category_id, url, title.
- 텍스트 파일 약 1~2MB 정도 규모(10,000건 수준)

### **구현할 기능**

- 기본: 단일 키워드 검색
- 심화: AND/OR 검색, 카테고리 필터, 스니펫 출력

### **인덱스 구성**

- Dictionary + Postings 형태의 역 인덱스 구성
- title과 내용을 형태소 분석(MeCab)하여 단어(term)를 추출
- term → 등장한 eid 목록(Postings)으로 연결

### **인터페이스**

- search.pl 키워드 형태로 실행
- URL + title 정도의 간단한 결과를 표시
- 옵션: category_id로 필터링

## **강의 28: 응답 사례와 사고방식**

### **전체 구조**

- **indexer.pl**
    - 데이터 읽기 → MeCab으로 단어 추출 → term별 eid 리스트 생성 → 파일로 저장
- **searcher.pl**
    - 저장된 인덱스 로딩 → 사용자 쿼리 입력 → 해당 term의 문서ID 반환

### **스니펫**

- 문서를 읽어서 키워드가 등장하는 근처 일부만 잘라 보여주는 간단한 방식

### **개선 포인트**

- 정식 AND/OR 쿼리 파싱으로 확장
- MeCab 대신 n-gram 기반 term 생성도 시도 가능
- Postings에 출현 위치까지 기록하면 더 정교한 검색 가능

### **Column: Twitter 스케일 전략 요약**

- Twitter도 기본적으로 MySQL + memcached + 파티셔닝 구조
- 핵심 아이디어는 데이터를 잘게 나눠 메모리에서 처리되도록 만든다.
