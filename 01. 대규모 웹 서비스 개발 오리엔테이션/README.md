# 1장. 대규모 웹 서비스 개발 오리엔테이션

## **강의 0: 이 책의 근간**

- 이 책은 하테나의 **대규모 데이터 처리 인턴십** 강의를 기반으로 구성되었다.
- 목표: 대규모 서비스를 운영하는 데 필요한 **기본 개념과 사고방식** 제공
- 다루는 핵심 주제:
    - 대규모 서비스 개발이란 무엇인가
    - 대규모 데이터 처리 개념
    - 캐시·스케일아웃·분산 처리 등 핵심 아키텍처 요소
    - RDBMS로 해결되지 않는 문제들
    - 운영 관점에서의 장애·성능 처리
- 다루지 않는 것:
    - 웹 개발 How-To, MVC/ORM 강좌, 언어 문법, 프레임워크 사용법
- 실전적인 **문제 해결 중심**의 접근, 실무에 필요한 관점 정립을 목표로 한다.

## **강의 1: 대규모 서비스와 소규모 서비스**

- 대규모 서비스가 갖는 특유의 문제와 구조적 요구사항을 설명한다.
- 소규모 서비스와의 가장 큰 차이:
    - **확장성 확보**(scale-out 중심), 부하분산 필요
    - **다중성 확보(고가용성)**
    - **효율적 운영 필요성**
    - **데이터 규모 증가에 따른 I/O 병목 문제**
- 하테나의 실제 서비스 규모 예시:
    - 월간 1500만 유저
    - 피크 430Mbps 트래픽
    - 수십 대~500대 서버 운영
- 대규모화는 단순히 서버 수를 늘리는 것이 아닌, **설계 사상**의 변화가 필요하다.

## **강의 2: 계속 성장하는 서비스와 대규모화의 벽**

- 서비스가 성장할수록 새로운 문제가 자연스럽게 발생한다.
- 하테나 초창기 사례:
    - 단일 서버 + 단순 구성으로 시작
    - 사용자 증가 → 트래픽 증가 → 장애 빈도 증가
    - 다중화, 부하 분산, 인프라 재구축 필요
- 문제의 핵심 원인들은 주로 다음에서 발생:
    - 네트워크 대역폭 부족
    - 서버·DB 부하
    - I/O 병목
    - 장애 처리 절차 미정비
- 성장 과정에서의 해결:
    - 데이터센터 이전
    - 서버 OS/구조 재정비
    - 캐시 활용 강화
    - 최소 비용으로 확장 가능한 구조 설계
- 계속 성장하는 서비스는 **비용 최소화 + 수용 능력 확보**의 균형이 필요하다.

## **강의 3: 서비스 개발의 현장**

- 하테나 내부의 실제 개발 방식·문화·도구를 소개하는 문단이다.
- 커뮤니케이션:
    - Wiki 기반 문서화·업무 조율
    - IRC 실시간 소통 (메신저인 듯)
    - 장애·운영 상황 공유
- 개발 프로세스:
    - 매일 아침 짧은 미팅
    - 담당자 중심의 태스크 진행
    - 테스트 코드 작성 → 리뷰 → 머지 → 배포
    - 상황에 따라 페어 프로그래밍 적용
- 사용 도구:
    - 언어: Perl 중심, C/C++, JS, PHP, Python, Ruby
    - 미들웨어: Linux, Apache, MySQL, memcached
    - 프레임워크: 자체 개발 Ridge
    - 에디터/OS: Linux + Emacs/Vim
    - 버전 관리: git + 자체 BTS
- 개발자에게 요구되는 역량:
    - 문제를 운영 관점에서 바라볼 수 있는 감각
    - 자신의 Workflow 이해
    - 팀 전체 품질·효율을 고려한 개발 습관
